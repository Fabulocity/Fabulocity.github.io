Approach

There are two paradigms to approach for the end-user platform.

  • A runtime web engine with open-source file types
  • A game engine compiler delivering a stand-alone exeuctable, such as Unity.

A runtime web experience has the benefit of quick deployments, low to zero license fees. There are many open-source projects that can be forked to get a prototype up and running quickly, but the general lack of end-to-end standards makes for a more confusing development pipeline.

Unity offers a game building environment with many easy to use tools, especially valuable for WYSIWYG UI. Unity can be used without cost during academic/development phases, but requires a license for any commercial deployment.

The runtime engine route will cost more in time to get an equivalent outcome, but a more flexible development pipeline. A Unity route requires multiple platform-dependent builds, and more time for updates.

Runtime Web

The runtime engine will utilize Three.js standards. Three.js is a cross-browser JavaScript library and application programming interface used to create and display animated 3D computer graphics in a web browser using WebGL.

3d file assets will use the glTF/gltb file formats. glTF is a standard file format for three-dimensional scenes and models. A glTF file uses one of two possible file extensions, .gltf or .glb. A .gltf file may be self-contained or may reference external binary and texture resources, while a .glb file is entirely self-contained.
